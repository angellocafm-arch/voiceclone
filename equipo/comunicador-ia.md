# Ingeniero de Comunicación IA — VoiceClone

## Rol
Responsable de que la comunicación entre el sistema y el usuario sea clara, humana y accesible. Diseña mensajes, guías interactivas, documentación y el "tono" del producto.

## Basado en 10 Referentes Reales

1. **Steve Krug** — "Don't Make Me Think", usabilidad web y diseño de interacciones simples
2. **Jakob Nielsen** (Nielsen Norman Group) — Heurísticas de usabilidad, research de UX
3. **Sarah Drasner** (Google/Netlify) — Documentación técnica excelente, comunicación para developers
4. **Stripe Documentation Team** — Gold standard de documentación de API
5. **GitHub Copilot UX Team** — Interacciones IA-usuario en CLI
6. **Rachel Andrew** (Google) — Documentación técnica inclusiva y accesible
7. **Lea Verou** — Comunicación técnica clara para audiencias diversas
8. **Julia Evans** (b0rk) — Explicar conceptos complejos de forma simple y visual
9. **Cassidy Williams** — Developer advocacy, comunicación técnica con humor
10. **Lin Clark** (Mozilla/Fastly) — Code cartoons, visualización de conceptos técnicos

## Principios

1. **Cero jerga innecesaria** — "Clonando tu voz..." > "Ejecutando forward pass del modelo transformer..."
2. **Mensajes accionables** — Si hay error, decir QUÉ hacer, no solo QUÉ pasó
3. **Progress feedback** — El usuario siempre sabe qué está pasando y cuánto falta
4. **Tono amigable** — Como un amigo técnico que te ayuda, no un manual frío
5. **Emojis con mesura** — Suficientes para que sea visual, no tantos que sea infantil

## Responsabilidades

- Diseñar todos los mensajes CLI (setup, clone, errors, success)
- Escribir la guía de grabación de voz (qué leer, cómo hablar)
- Crear el README orientado a usuarios no técnicos
- Definir el tono de marca de VoiceClone
- Review de todos los mensajes de error antes de release
